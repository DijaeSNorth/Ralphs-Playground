Priority Fix Reference - Menus, Customization, Movement

Scope:
- Fixed Crew, RepDex, Goals, and Settings menu access without adding new gameplay systems.
- Moved the gameplay menu to a compact top-right grid so it no longer covers movement instructions, touch controls, target preview, or stamina.
- Restored title-screen character customization with a clear Customize Character button.
- Added Front View and Back View controls to the existing character preview, alongside Rotate Left, Rotate Right, and Reset View.
- Reused the Settings panel to reopen customization after gameplay has started.

Movement:
- Keyboard input is screen-relative for the fixed 2.5D camera: W/Up moves visually upward, S/Down moves downward, A/Left moves left, and D/Right moves right.
- Pointer movement now supports click/tap-and-hold on the playfield, using the screen position relative to the canvas center.
- UI pointer events remain separate from playfield movement.

Files touched:
- src/ui/hud.ts
- src/game/input/actions.ts
- src/render/app/createGymBuddyGame.ts
- src/style.css
- docs/priority-fix-reference.txt

Follow-up input priority fix:
- Added an interactive-target guard in src/game/input/actions.ts so playfield click/tap movement ignores buttons, panels, HUD, target preview, capture actions, settings, crew, RepDex, goals, tutorial cards, and modal-like UI.
- Added InputController.queueCapture() and cancelPointerMove() so UI capture actions stop any active click-to-move gesture before queueing capture.
- Converted the target preview "Arm Wrestle" affordance into a real button with data-capture-action.
- Connected HUD capture action events to the existing input capture queue in createGymBuddyGame.ts.
- Pointer movement now starts only when the game has started and HUD interaction/capture/customization overlays are not active.
- CSS now explicitly keeps visible HUD controls above the canvas and disables pointer events for hidden panels.

Customization fix:
- Player mesh creation now reads real PlayerAppearance values instead of legacy numeric color hints.
- Skin tone, hair style, muscle build, frame, height, shoulders, torso, arms, legs, chest, wings/back, glutes, thighs, and calves all feed the rendered player mesh.
- Appearance listeners now hydrate immediately, so the preview and gameplay player receive the current appearance on startup and update live from Settings customization.
- Creator controls include short helper descriptions and a live Build/Frame/Focus summary.
- Front View, Back View, Rotate Left, Rotate Right, and Reset View continue to drive the existing preview rotation listener.

Player character redesign:
- Replaced the harsh blocky player mesh with a softer 2.5D retro trainer silhouette.
- The player now has a larger rounded head, simple eyes/smile, headband, gloves, sneakers, trainer outfit, rear backpack/backplate details, and a cleaner shadow blob.
- Customization still drives skin, hair, muscle build, frame, shoulders, arms, chest, wings/back, torso, legs, glutes, thighs, calves, and height.
- Slider ranges are visually clamped in the mesh so high values read heroic/swole without extreme distortion.

Customization polish:
- Default player appearance is now warm skin, swept-back hair, athletic build, balanced frame, and lightly boosted heroic proportions.
- Build styles are tuned from Beginner through Elite with intentional mass, arm, chest, and leg scaling.
- Frame styles now add tasteful shoulder/waist/hip/height/lower-body differences without extreme distortion.
- Live creator summary now reports Build, Frame, Base, and a readable visual Focus such as boosted shoulders or heroic silhouette.
